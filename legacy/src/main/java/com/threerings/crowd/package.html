Builds on the Presents framework to provide services uesful for chatting and moving from place to place in a distributed space.

The main Crowd concepts are that of Body and Place and the notion of a PlaceManager on the server and a pairing of PlaceController and PlaceView on the client.

  • client - the client side of the Crowd services.
  • data - classes that are shared between client and server.
  • server - the server side of the Crowd services.

Crowd also provides a mechanism for Chat in Places and between individuals that are logged on.

  • client - the client side of the Crowd Chat services.
  • data - classes that are shared between client and server.
  • server - the server side of the Crowd Chat services.

Crowd also integrates with the Presents peer system to make its Chat services work in a peer environment.

  • peer.client - the client side of the Crowd Peer Chat services.
  • peer.data - classes that are shared between servers.
  • peer.server - the server side of the Crowd Peer Chat services.
